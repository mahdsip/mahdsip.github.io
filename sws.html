Servicios Web Semánticos

 

Se trata de permitir a las maquinas interpretar el significado de los datos en base a anotaciones semanticas basadas en descripciones semanticas.

 

 

La estructura que se utiliza para el modelo de datos suele ser RDF, que describe tripletas de manera sujeto-predicado-objeto, por ejemplo:

PaginaWikipediaMadrid-titulo-wikimadrid

PaginaWikipediaMadrid-autor-Mahuerta.

 

 

El continente de ls datos normalemente suele ser XML, de manera que lo anterior quedaria :

<PaginaWikipediaMadrid>

<titulo>wikimadrid</titulo>

<autor>MAhuerta</titulo>

</PaginaWikipediaMadrid>

Ademas de esta combinacion RDF-XML existen tambien N2, Turtle, Ntriples.

 

El formato de los datos, en lugar de ser texto libre se utiliza URI(esquema:autoridad/ruta?clave=valor#fragmento) . Hay que recordar que una URI se distingue de una URL que las URL pueden aparecer y desaparecer en el tiempo, ademas de que no permite fragmentos.

 

De nuevo, la informacion quedaria de la siguiente manera (XML+namespace) :

<rdf:RDF

xlmns:rdf=”http://urlRDF”

xlmns:dc=”http://urlDC”

>

<rdf:description rdf:about =”http:es.wikipedia.org/Madrid”>

<dc:Title>wikiMadrid</Title>

<dc:autor>MAhuerta</Mahuerta>

</description>

</RDF>

 

Por ejemplo, hay una extension de RDF llamada RDFs para definir clases y propiedades

rdf:Class, rdf:property, rdf:definedBy,etc...

 

La terminologia para definir la ontologia Web (lo equivalente al namespace dc en el ejemplo) descrita por la W3C es OWL, que tiene como antecedente DAM+OIL. La pila quedaria de la siguiente forma:

 

OWL = RDF + (XML+namespace) + URI

 

 

 

Este es el esquema semantico para definir los servicios web, ya tendremos la estructura para definirlos. Existen multiples “implementaciones” concretas como son:

 

OWL-S

 

Esta compuesto por tres partes:

Service profile → ¿Que ofrece?

Service grounding → ¿Como se definiria (SOAP,RPC,etc)

Service Model → ¿Como funciona?

 

WSMO

 

Esta descrito por:

Ontologias (Las ontologias utilizadas)

Mediadores (conectividad entre distintas ontologias)

Objetivos (objetivos del cliente cuando consume el servicio)

Servicio web (descripcion semantica)

 

WSDL-S y SWSF (estos ultimos menos utlizados).

 

El problema es que estas descripciones de servicio son muy pesadas para inferir sobre ellas a dia de hoy. Por ello se simplifican al maximo y se hacen versiones ligeras, como WSMO-Lite o MicroWSMO. Lo mas que se puede simplificar estas ultimas es el demonimando minimal service model (MSM) , que se corresponden respectivamente con POSM (Orientado a procedimiento) y ROSM( orientado a recurso) .

Para la interaccion con web services es esencial su complementacion con WSDL. Al no estar WSDL preparado para contener anotaciones POSM, integramos anotaciones con SA-WSDL

SA-WSDL

ModelReference

lifting squema mapping

 

Siguiendo este modelo, se puede hacer un mapeo facil de las anotaciones SA-WSDL a POSM que ya si es RDF de manera que podamos utilizar una base de datos de inferencia semantica y utilizando el lenguaje estandar definido por la W3C SPARQL para realizar consultas semanticas.
