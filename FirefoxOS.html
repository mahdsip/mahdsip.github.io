Lo primero que me propuse fue compilar yo mismo la última versión 2.0 por aquel entonces, ya que los amiguetes de Movistar no pasaron de la versión 1.1 nunca... La verdad es que las páginas de referencia de Firefox están bastante bien, aunque el proceso es un poco tedioso y facilmente se pueden obtener errores de configuración. En mi caso, lo único que me funcionó fue crearme una máquina virtual sobre un Güindous (para evitar complicaciones con conectar el móvil por USB ya que es necesario para la compilación ¿WTF!!???)con Ubuntu 12.04 que es el entorno que recomiendan.

Guía para colaborar con proyectos open source en Github

A continuación os dejo una pequeña guía para quien quiera colaborar no solo con este proyecto sino con cualquier proyecto open source alojado en github

  • Abrir una cuenta en GitHub, ahí se alojarán todos los forks que hagas de otros proyectos así como tus propios cambios.

  • Hacer un fork en nuestra cuenta de github del proyecto en el que queramos colaborar.

  • Una vez tenemos el fork en nuestra cuenta, descargarlo a la maquina local con "git clone" el fork.

Una vez descargado, crear una rama local en el proyecto para añadir las modificaciones, con "branch " por ejemplo, de esta manera tendremos el fork inicial siempre intacto. Cuando los cambios estén listos para subir a GitHub, subirlos al repositorio remoto de nuestro fork con git push origin . Abrir la sección Pull Requests en GitHub y crear un pull request a la rama master del proyecto original, que será en este caso el proyecto con el que queremos colaborar. Cuando suba mi solución al repositorio original, para poder añadir a la rama master de vuestro fork los cambios de la rama master del repositorio original al que hemos hecho el pull request es necesario realizar los siguientes pasos:

  • Copiar la URL del repositorio original en el que queremos colaborar

  • Añadir un nuevo repositorio remoto con: git remote add principal . Después de esto tendréis dos repositorios remotos: origin (vuestro propio repositorio -el fork- en GitHub) y principal (el repositorio original del proyecto en el que quieres colaborar). Esto se puede ver con git remote -v

  • Descargar los cambios del repositorio principal con: git fetch principal

  • Situarse en vuestra rama master con: git checkout master y finalmente añadir los cambios de la rama master del repositorio principal: git merge principal master

  • Ojo, aquellos que hubieran realizado modificaciones (commits) en la rama master del fork tendrán algún problema de conflictos. Por eso es importante realizar siempre los cambios en una rama específica de vuestro fork. De esa forma “dejáis libre” la rama master para sincronizarla con la rama master del repositorio original.
