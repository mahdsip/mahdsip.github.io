XML

eXtended Markup Language

Es el lenguaje de etiquetas que permite utlizar un formato y continente de mensaje estandar e interoperable. Se trata de un subconjunto del lenguaje SGML, saltándose algunas de sus restricciones. Para que un XML sea correcto tiene que se valido (que cumpla todas las normas morfologicas de un documento XML y bien formado (que cumpla con las normassintacticas definidas en su esquema).

 

XSD

Para definir la estructura del mensaje se pueden usar varias tecnlogias con DTD, pero en este caso concreto no se permite por ejemplo mezclar estructuras de distintas fuentes, por lo que generalmetne se utiliza el XDS que es mas rico en este aspecto. La programación en Schema XML se basa en namespaces o espacios de nombres Podemos encontrar una analogía entre éstos y los llamados packages en Java. Cada namespace contiene elementos y atributos que están estrechamente relacionados con el namespace. Así, a la hora de definir un elemento o un atributo de un namespace, siempre se creará una conexión entre los diferentes campos de éste. Además, esta forma de trabajar nos permite relacionar elementos que no están en el mismo namespace.

 

<?xml version="1.0" encoding="UTF-8"?>
								<xsd:schema xmlns:xsd="http://www.w3.org/2001/XMLSchema">
								     <xsd:element name="Libro">
								         <xsd:complexType>
								            <xsd:sequence>
								                 <xsd:element name="Título" type="xsd:string"/>
								                 <xsd:element name="Autores" type="xsd:string" maxOccurs="10"/>
								                 <xsd:element name="Editorial" type="xsd:string"/>
								            </xsd:sequence>
								            <xsd:attribute name="precio" type="xsd:double"/>
								        </xsd:complexType>
								   </xsd:element>
								</xsd:schema>

XSL

 

El Lenguaje de Hoja de Estilo Extensible (eXtensible Stylesheet Language, XSL) es una familia de lenguajes que permiten describir como los archivos codificados en XML serán formateados (para mostrarlos) o transformados. Hay tres lenguajes en esta familia:

XSL Transformations (XSLT), que permite convertir documentos XML de una sintaxis a otra (por ejemplo, de un XML a otro o a un documento HTML). El resultado puede ser o no un XML

Las templates se aplican en orden secuencial a cómo están definidas en el apply-templates. Si al recorrer un nodo coinciden 2 templates se elige una sola o bien estableciendo una prioridad o si no será la más específica. Para aplicar templates a nodos hijos hay que poner dentro de la template padre <xsl:apply-templates>, con select = “//*” buscas en todos sus nodos.

 

<xsl:template match=”/”>

<xsl:apply-templates select =”//*”>

<xsl:apply-templates select =”//*” mode=”2”>

</xsl:template>

<xsl:template match =”*”>

<hola>

<xsl:value-of select = “name()”/>

</hola>

</xsl:template>

<xsl:template match = “*” mode = “2”>

<hola2>

<xsl:value-of select =”name()”/>

</hola2>

</xsl:template>

 

Esto aplica al mensaje las dos templates una seguida de la otra. Se puede acceder a los atributos <xsl:value-of select =”@node”> y se le pueden aplicar templates igual <xsl:apply-templates select = “@node”>

Si en lugar de ir nodo por nodo viendo si coincide un patron quiero recorrer todo el documento primero aplicando un patron y luego que contiene se hace el <xsl:for each>

Se pueden introducir valores en el XML con <xsl:element name =”” > asi como atributos <xsl:attributes name =””>

Copy→ copia solo el nodo, copy-of lo copia con todos sus hijos.

En el mundo JAVA, para parsear mensajes XML en objetos existe la espcificacion JAXP. Existen otras dos especificaciones que son independientes del lenguaje de programacion que son SAX para acceder unicamente a partes del documento y DOM que crea un arbol completo del documenteo en memoria, JAXP cumple con ambas especificaciones. Implementaciones en Java de SAX se encuentra SAXON y como implementacion DOM se encuentra Xerces. Ademas existe una implementacion unicamente pensada para java llamada JDOM para manipular los elementos mas facilmente (tan facil como sacarlos y meterlos en un list()).

 

Por ultimo existe otra teclogia en Java para hacer marshaling y unmarshaling de documentos XML en objetos bajo el estandar JAXB. Esta especificacion te permite mapear directamente objetos java en documentos XML en base al esquema XSD definido. En java existen dos implementaciones la propia de de Java y JAXME. Por ultimo existe la tecnologia XMLBeans que no sigue el estandar JAXB pero realiza la misma funcion.
