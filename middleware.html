Middleware

Es un software intermedio entre el cliente y el servidor que permite gestionar diversos mecanismos dentro de la comunicación. Esconde la naturaleza distribuida de las aplicaciones, esconde las especificaciones de plataforma, HW, lenguaje, protocolos, etc.. proporcionando una interfaz común para interorperar además de una serie de funcionalidades comunes a las aplicaciones que las utilizan desplazando la responsabilidad de esa funcionalidad de las aplicaciones al sistema intermedio.

 

Se puede hablar de dos tipo de middleware:

 

Middleware base: Lo requieren todos los clientes y servidores. Se pueden considerar dentro de esta clasificación toda la pila OSI de comunicaciones (TCP/IP,etc..) . Da soporte a la construcción de middleware sobre ellos.

 

Middleware de applicaciones: Proporciona una funcionalidad a nivel de aplicación a los extremos que participan en la comunicación. Existen los sig

 

  • Middleware transaccional: Procesan trasacciones y se aseguran de la transaccion se ha realizado correctamente. Son los monitores de teleproceso (CICS), o productos como Tuxedo o Encina.

  • Middleware de BBDD: Reducen la complejidad de acceso a la base de datos. En esta categoria entrarian tecnologias como JDBC, ODBC.

  • Middleware de aplicaciones web: Es el sofware intermedio entre el cliente y el servidor que proporciona el acceso a la aplicación web. Entrarian las siguientes tecnologias:

    CGI → Programas que se ejecutan en el servidor y pueden actuar como pasarela con una app, con una BBDD o generear un documento HTML independientemente del S.O. Un programa CGI puede ser escrito en cualquier lenguaje de programación que produzca un archivo ejecutable, el lenguaje mas popular para las Cgi es PERL. Cada peticion HTTP ejecuta un proceso, analiza la solicitud y genera un resultado. Fue de las primeras tecnologias en proporcionar contenido dinamico a las paginas web.

     

    SERVLET → Pequeños programas en JAVA que ejecutan de forma persistente con una activacion muy rapida, procesan la peticion y generan la respuesta.

     

    ASP→ Fichero de texto con secuencias de comandos junto con el HTML necesario. Al ser llamado por un navegador el motor ASP del IIS lo ejecuta y envia la salida al Navegador que lo invoca.

     

    JSP → Pequeños trozos de codigo en JAVA que se insertan dentro de las paginas WEB y al ejecutarse en el servidor generan un resultado en HTML. Son independientes del SO y del procesador.

     

    PHP → Lenguaje cuyos programas se insertan tambien dentro de las paginas web muy rapido de implantar y desarrollar, proporciona acceso muy simple a la base de datos.

     

  • Middleware de intergracion:Se trata de integrar elementos pertenecientes a un negocio que pueden ser sistemas (ej:departamentos) o elementos creados para el proyecto de integración (ej:servidor LDAP para autenticacion de usuarios). En los nuevos escenarios de integración son claves los procesos de negocio (BPM) en los que la resolución de un problema requiere la intervencion coordinada de varios sistemas.

    Existen dos tecnologias para dar soporte al Middleware de integración de BP:

     

    Motor de integración: Ofrece un servicio para la definición y control de procesos de negocio respondiendo a problemas de integración a escala corporativa. Se basa en el enrutado de mensajes basado en el contenido y se le suele dar el nombre de MOM(Message Oriented Middleware). Existen una serie de patrones de integración denominados EIP con distintas operativas de integración. Apache Cammel es la implementación más famosa de EIP.

     

    ESB: Bus de integración empresarial diseñado para superar las limitaciones de los Mint y responder a las necesidades de integración de empresas a escala superior. Los ESB pueden estar físicamente distribuídos en distintos segmentos pero todos ellos son una misma entidad lógica. Dispone de muchas más funcionalidades adicionales al enrutado de mensajes en base al contenido como validación y transformación de la información, definición y control de procesos de negocio , autenticación, autorización, monitorización,etc...
