Middleware

It is a software between the client and the server that manages various mechanisms within the communication. Hides the distributed nature of applications, hidden platform specifications, HW, language, protocols, etc .. providing a common interface for interorperar plus a range of common functionality to applications that use shifting the responsibility for that functionality intermediate system applications.

 

One can speak of two types of middleware:

 

Based Middleware: What require all clients and servers. They can be considered within this classification entire OSI communications (TCP / IP, etc ..) stack. Supports middleware building upon them.

 

Middleware Applications: Provides application-level functionality to extremes involved in communication. There sig

 

    Transactional middleware: Process trasacciones and ensure the transaction was successful. Teleprocessing monitors are (CICS), or products such as Tuxedo and Encina.

    Middleware DB: Reduce the complexity of access to the database. In this category they would enter technologies such as JDBC, ODBC.

    Middleware Web applications: It is the intermediate software between the client and the server that provides access to the web application. Enter the following technologies:

    CGI → Programs running on the server and can act as a gateway with an app with a DB or an HTML document generear regardless of SO A CGI program can be written in any programming language that produces an executable file, the most popular language for CGI is PERL. Each HTTP request executes a process, analyzes the request and generates a result. It was among the first technologies to provide dynamic content to web pages.

     

    SERVLET → Small Java programs that run persistently with a very rapid activation, process the request and generate the answer.

     

    File → ASP scripted text with the necessary HTML. When called by a browser the ASP engine of IIS executes and sends the output to the browser invoking it.

     

    JSP → Small pieces of Java code that are inserted into Web pages and run on the server to generate a result in HTML. They are independent of the OS and processor.

     

    → PHP language whose programs are also inserted into web pages very fast to implement and develop, provides simple to access the database.

     
    Intergration Middleware: This involves integrating elements belonging to a business that can be systems (eg departments) or elements created for the project of integration (eg LDAP server for user authentication). In the new integration scenarios are key business processes (BPM) where the problem resolution requires the coordinated intervention of various systems.

    There are two technologies to support integration middleware BP:

     

    Integration Engine: Provides a service for the definition and control of business processes responding to integration problems at the corporate level. It is based on the routing of messages based on the content and is usually called MOM (Message Oriented Middleware). There are several known PID integration patterns with different operational integration. Cammel Apache is the most famous of EIP implementation.

     

    ESB: Enterprise Integration Bus designed to overcome the limitations of the Mint and respond to the needs of enterprise integration to a larger scale. The ESB can be physically distributed in different segments but they are one logical entity. It has many more additional features to the routing of messages based on content as validation and transformation of information, definition and control of business processes, authentication, authorization, monitoring, etc ...
