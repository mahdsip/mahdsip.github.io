RaspberryPi

Playing with raspberry pi model B

My main goal was to create a private cloud with which to synchronize my phone without having to rely on third party services. There is a formidable open source software allows me to create this easily called owncloud, and with whom you can have files, contacts, calendar, notes and tasks synchronized with your own server. This sounds perfect to "unplug from the Matrix" so these are the steps I followed

  • Insert sd card into the computer
  • lsblk to see what is the volume name (usually sdb if no volume connected to your computer.
  • download the latest version of raspbian
  • record the image on the sd
  • sudo dd if = <path-to-image> <name-of-the-image> of = / dev / <device-sd>
  • Recollect the zip and unzip it .img file which flaseeis

You receive a warning that they have spent the 3.3GB approx.



Now we just turn on the raspberry and make the initial tasks, the most important partition to expand all available memory, change the password and enable ssh and be able to do everything remotely.

Assign fixed ip
 

  • $ Sudo nano / etc / network / interfaces


# Configure fixed IP address for eth0
auto eth0
iface eth0 inet static
address 192.168.1.50
netmask 255.255.255.0
192.168.1.0 network
broadcast 192.168.1.255
gateway 192.168.1.1
 

  •  sudo /etc/init.d/networking restart

 
$ Sudo ifconfig eth0 down
$ Sudo ifconfig eth0 up

  • Install no-ip client to automatically update my IP

    cd / usr / local / src /
    wget http://www.no-ip.com/client/linux/noip-duc-linux.tar.gz
    tar xf noip-duc-linux.tar.gz
    noip-2.1.9-1 cd /
    make install

 

  • Create certificates

sudo apt-get install openssl
ssl sudo a2enmod
a2enmod rewrite sudo
service apache2 restart

 

It is possible to use the default cert that comes with apache
a2ensite default-ssl
service apache2 restart

or create your own certificate

  • Generate Certification Body

sudo openssl req -new> new.ssl.csr
 

  • Generate the certificate


sudo openssl rsa -in -out privkey.pem new.cert.key
 

  • Sign the certificate


sudo openssl x509 -in -out new.ssl.csr new.cert.cert -req -signkey new.cert.key -days 3650
 

  • Copy the certificates to the default folder


sudo cp new.cert.cert /etc/ssl/certs/server.crt
sudo cp new.cert.key /etc/ssl/private/server.key

Ensure that server.key has only read permissions.


 

  • Install owncloud
  • add repo


echo 'deb http://download.opensuse.org/repositories/isv:/ownCloud:/community/Debian_8.0/ /' >> /etc/apt/sources.list.d/owncloud.list


cd / tmp
wget http://download.opensuse.org/repositories/isv:ownCloud:community/Debian_8.0/Release.key
apt-key add - <Release.key

apt-get update
apt-get install owncloud
 

  • Create a table and mysql user on for owncloud


mysql --defaults-file = / etc / mysql / debian.cnf

Enter the following

CREATE DATABASE owncloud;
CREATE USER owncloud @ localhost IDENTIFIED BY 'mysecurepassword';
GRANT ALL PRIVILEGES ON * TO owncloud owncloud @ localhost.;
flush privileges;
quit

mkdir / var / owncloud
chown www-data: www-data / var / owncloud
chmod 750 / var / owncloud
 

  • Enter the administrator username and password in the login fields. Please choose a secure password and a username Also That Is not "admin" or "administrator" might be a good choice to make it less easy for attackers to guess your admin login.

 

  • I dont like my private files to be stored in the website root / var / www, so I choose the folder / var / owncloud here.

 

  • OwnCloud is using sqlite as storage engine by default. This is not a good choice performace wise, so I will choose MySQL database as backend. We Have a MySQL database created above, enter the details of That database now:


    Username: owncloud
    Password: The password that you 'Have choosen for the database.
    Database name: owncloud
    Hostname: localhost

 

Just in case you havent decided to use apache cert that comes by default
Redirect requests to port 443:

nano /etc/apache2/conf-enabled/owncloud.conf

add

443 #Listen
#uncomment and change the above only for other ports
 
 <VirtualHost 192.168.0.16:80>
RewriteEngine on
SERVER_PORT RewriteCond% {}! ^ $ 443
RewriteRule ^ / (. *) Https: //% {HTTP_HOST} / $ 1 [CN, R, L]
</ VirtualHost>
 
<VirtualHost 192.168.0.16:443>
SSLEngine on
SSLCertificateFile /etc/apache2/ssl/privkey.pem
SSLCertificateKeyFile /etc/apache2/ssl/private/server.key
DocumentRoot / var / www / owncloud /
 
Alias ​​/ owncloud "/ var / www / owncloud /"
<Directory "/ var / www / owncloud">
    Options + FollowSymLinks
    AllowOverride All
 
    <IfModule mod_dav.c>
      Dav off
    </ IfModule>
 
    SetEnv HOME / var / www / owncloud
    SetEnv HTTP_HOME / var / www / owncloud
</ Directory>
 
<Directory "/ var / www / owncloud / data /">
  # Just in case if .htaccess gets disabled
  Require all denied
</ Directory>
</ VirtualHost>

# service apache2 restart

Force all connections are https:
nano /etc/apache2/conf-enabled/owncloud.conf

add
 

<VirtualHost *: 443>
   ServerName cloud.owncloud.com
   Header always add Strict-Transport-Security "max-age = 15768000"
</ VirtualHost>
