Semantic Web Services

 

It's allow machines to interpret the meaning of data based on semantic annotations based on semantic descriptions.

 

 

The structure used for the data model is usually RDF triples describing such subject-predicate-object, for example:

PaginaWikipediaMadrid-titulo-wikimadrid

PaginaWikipediaMadrid-autor-Mahuerta.

 

 

The continent of the data it is normally XML

<PaginaWikipediaMadrid>

<titulo>wikimadrid</titulo>

<autor>MAhuerta</titulo>

</PaginaWikipediaMadrid>

Besides RDF-XML there exits also N2, Turtle, Ntriples.

 

The data format, instead of free text it is URI(esquema:autoridad/ruta?clave=valor#fragmento) . Hay que recordar que una URI se distingue de una URL que las URL pueden aparecer y desaparecer en el tiempo, ademas de que no permite fragmentos.

 

XML+namespace :

<rdf:RDF

xlmns:rdf=”http://urlRDF”

xlmns:dc=”http://urlDC”

>

<rdf:description rdf:about =”http:es.wikipedia.org/Madrid”>

<dc:Title>wikiMadrid</Title>

<dc:autor>MAhuerta</Mahuerta>

</description>

</RDF>

 

For example there is a RDF extension called RDFs to define classes and properties

rdf:Class, rdf:property, rdf:definedBy,etc...

 

The terminology to define the Web ontology (the equivalent of the dc namespace in the example) is described by the W3C OWL, which has been preceded DAM + OIL. The stack would be as follows:

 

OWL = RDF + (XML+namespace) + URI

 

 

 

This is the schema to define semantic web services, we will have to define the structure. There are multiple implementations; concrete such as:

 

OWL-S

 

It has 3 parts:

Service profile → ¿Que ofrece?

Service grounding → ¿Como se definiria (SOAP,RPC,etc)

Service Model → ¿Como funciona?

The most that can simplify these last demonimando service is minimal model (MSM), which correspond respectively to POSM (process oriented) and ROSM (resource-oriented).
