Architectural styles

Although now I have a few years working on data mining, during my career I have had the privilege of working with different architectures such as SOA, BPM, Cloud Computing or JAVA.

There have been and there are many ways to classify architectural styles, difenciables architectural patterns that are purely theoretical as these are purely practical and can even mix several of these in a single architecture.

 

Style data stream:

    Pipes and filters (transformation of a data stream with a final result)
    Batch (batch and scripts)

Data-centric styles:

    Board or repository
    DB
    Hipertextos

Call and return Styles:

    Layers (MVC)
    Object Orientation
    Component

Mobile code styles:

    Virtual machines (compiled code)
    Interpreters (interpreted code)

Heterogeneous styles:

    Process control
    Based on attributes (reasoning about attributes)

Peer2Peer styles:

    Based on events (Invoking a function when a Publisher / Susbcriber event occurs)
    SOA a call is made to a service, passing the necessary parameters and is expected resultado.SOA2.0 = SOA + events.
    Based on resources. The resorce can be discovered through a URI and get your state to operate with the resource.
