Servicios Web

Implementación SOA

Se trata de una implementación de un arquitectura SOA de tipo archivo en la que intervienen los siguiente elementos:

 

 

Protocolo para el intercambio de mensajes, tipicamente SOAP

Un continente de mensaje estandar interoperable, tipicamente XML

Uso de un lenguaje estandar para definir el contrato del servicio, tipicamente WSDL.

 

Esto en la practica, se traduce como una peticion, generalmente con el protocolo de comunicación HTTP Post, pero no necesariamente, podría tratarse otros protocolos de comunicación como JMS o SMTP aunque HTTP se ha convertido en el estandar de facto. El continente del mensaje suele ser típicamente también un mensaje XML, aunque existen otras posibilidades como JSON. Además se puede contar con una entidad intermedia llamada registro UDDI que contiene todos los servicios web disponibles con sus correspondientes WSDLs de manera que cuando un servicio web necesite descubrir un servicio le haga una petición a este registro para que le devuelva el servicio web que cumple con sus necesidades y las condiciones de servicio en el contrato establecido en el WSDL pudiéndose incluso convertir en un API Manager con funcionalidades añadidas como gestión de seguridad o de versiones del servicio.

 

Por tanto, típicamente, los servicios web se componen de:

 

WS = SOAP + XML + WSDL + UDDI

 

En el mundo Java, desde Java 6 existe la especificación estándar para servicios web llamada JAX-WS, asi como su implementación. Además existen diversos productos adicionales que proporcionan funcionalidades adicionales a esta especificación con son Apache Axis y Apache CXF.

 

WSDL

 

Es el lenguaje para definir un servicio web que expone un servicio empresarial de grano grueso que encapsula diferentes operaciones, basándolo todo sobre XML. Describe una interfaz publica a los servicios web. En el se definen las operaciones que se pueden demandar del servicio web y los datos que se requieren. El formato de los datos se pueden especificar a traves de XSD y el protocolo para hacer la peticion normalmente es SOAP

 

Existen dos patrones para acceder al servicio:

RPC: El mensaje contiene un elemento con el nombre del metodo que se quiere invocar y a continuacion los parametros y valores. Al llegar la peticion se mapeara directamente con la clase correspondiente devolviendo un valor. Existen a su vez dos tipos, encoded que es codigo altamente acoplado de manera que si cambia algun detalle de la interfaz tienes que cambiar esto tambien, es la manera más correcta de implementar polimorfismo y Literal, en el que no se hacen comprobaciones sobre la interfaz y se envian los parametros y valores de manera “literal”.

 

Document: No existe un elemento envoltorio del metodo como en el anterior, las partes del mensaje aparecen directamente dentro del cuerpo del mensaje. Existentambien dos tipos, Literal en el que el mensaje concuerda exactamente con el esquema existente y Literal Wrapped en el que le indicas una etiqueta por encima para saber la operación que quieres realizar y luego todo el mensaje.

 

 

<definitions name="StockQuote"
								          targetNamespace="http://example.com/stockquote.wsdl"
								          xmlns:tns="http://example.com/stockquote.wsdl"
								          xmlns:xsd1="http://example.com/stockquote.xsd"
								          xmlns:soap="http://schemas.xmlsoap.org/wsdl/soap/"
								          xmlns="http://schemas.xmlsoap.org/wsdl/">

								    <types>
								       <schema targetNamespace="http://example.com/stockquote.xsd"
								              xmlns="http://www.w3.org/2000/10/XMLSchema">
								           <element name="TradePriceRequest">
								              <complexType>
								                  <all>
								                      <element name="tickerSymbol" type="string"/>
								                  </all>
								              </complexType>
								           </element>
								           <element name="TradePrice">
								              <complexType>
								                  <all>
								                      <element name="price" type="float"/>
								                  </all>
								              </complexType>
								           </element>
								       </schema>
								    </types>

								    <message name="GetLastTradePriceInput">
								        <part name="body" element="xsd1:TradePriceRequest"/>
								    </message>

								    <message name="GetLastTradePriceOutput">
								        <part name="body" element="xsd1:TradePrice"/>
								    </message>

								    <portType name="StockQuotePortType">
								        <operation name="GetLastTradePrice">
								           <input message="tns:GetLastTradePriceInput"/>
								           <output message="tns:GetLastTradePriceOutput"/>
								        </operation>
								    </portType>

								    <binding name="StockQuoteSoapBinding" type="tns:StockQuotePortType">
								        <soap:binding style="document" transport="http://schemas.xmlsoap.org/soap/http"/>
								        <operation name="GetLastTradePrice">
								           <soap:operation soapAction="http://example.com/GetLastTradePrice"/>
								           <input>
								               <soap:body use="literal"/>
								           </input>
								           <output>
								               <soap:body use="literal"/>
								           </output>
								        </operation>
								    </binding>

								    <service name="StockQuoteService">
								        <documentation>My first service</documentation>
								        <port name="StockQuotePort" binding="tns:StockQuoteSoapBinding">
								           <soap:address location="http://example.com/stockquote"/>
								        </port>
								    </service>

SOAP

 

Es un protocolo de comunicación para hacer peticiones a servicios web y recibir un resultado intercambiando ficheros. El mensaje del protocolo SOAP se compone de:

Envelope: Siempre es el primer elemento del mensaje

Header: Permite enviar información relativa a como debe ser procesado el mensaje.

Body: El propio contenido del mensaje en el que si el tipo del WSDL es orientado a RPC aparecerán los detalles del método al que se llama y si está orientado a documento aparecerá directamente el mensaje.
